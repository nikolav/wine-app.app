non-functional testing

======================

- stability, pressure performance and app response/processing time

- reliebility under usage peeks, concurrent user volume

- verifing critical, app break points

- in non-func. requirements (nfr.) customer specifies load requirements

- users have expectations requirements; does it respond fast

- in dev. concurrent usage is simulated with threads

- automation tools, load runners

- black box testing technique


testing types:

  1. load

- response load time, in s, for N concurrent users; (app requirement; <4s for 5000 concurrent users)

  2. stress

- break circumstances; how much load can take before crash; load ovrload response times; how many requests it can take the most before break

  3. soak

- continous load volume; runing test for extended period of time with the peek load

  4. volume

- testing db performance for edge cases; db overload for crash points

  5. spike

- test sudden user spikes;
